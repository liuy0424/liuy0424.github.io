对表进行聚合查询

聚合函数

常用聚合函数：

  • COUNT: 计算表中的记录数（行数）
  • SUM: 计算表中数值列中数据的合计值
  • AVG: 计算表中数值列中数据的平均值
  • MAX: 求出表中任意列中数据的最大值
  • MIN: 求出表中任意列中数据的最小值

    计算表中数据的行数

    示例： 计算全部数据的行数
    SELECT COUNT(*)
  FROM product;

    计算NULL之外的数据的行数

    示例：计算NULL之外的数据行数
    SELECT COUNT(purchase_price)
  FROM product;

    注：
    COUNT 函数的结果根据参数的不同而不同。
    COUNT(*) 会得到包含NULL的数据行数，而COUNT(<列名>)会得到NULL之外的数据行数

计算合计值

示例： 计算销售单价和进货单价的合计值

SELECT SUM(sale_price), SUM(purchase_price)
    FROM product;

注：聚合函数会将NULL 排除在外。只有COUNT(*)除外。

计算平均值

示例：计算销售单价平均值

SELECT AVG(sale_price), AVG(purchase_price)
    FROM product;

注： sale_price的平均值是总额除以8， purchase_price的平均值是总额除以6.

计算最大值和最小值

示例： 计算销售单价的最大值和进货单价的最小值

SELECT MAX(sale_price), MIN(purchase_price)
    FROM product;

MAX 和 MIN 函数也可以用于其它类型的数据，比如日期型。例如：

SELECT MAX(regist_date), MIN(regist_date)
    FROM product;

注： MAX/MIN 函数几乎适用于所有数据类型的列。 SUM/AVG 函数只适用于数值类型的列。

使用聚合函数删除重复值

示例：计算去除重复数据后的数据行数

SELECT count(DISTINCT product_type)
    FROM product;

注：
想要计算值的种类时， 可以在COUNT 函数的参数中使用 DISTINCT。
所有聚合函数都可以使用 DISTINCT，不仅是COUNT。

对表进行分组

GROUP BY 子句

语法结构：

SELECT <列名1>, <列名2>, ...
    FROM <表名>
    GROUP BY <列名1>, <列名2>,...;

示例： 按照商品种类统计数据行数

SELECT product_type, COUNT(*)
    FROM product
    GROUP BY product_type;

注1： GROUP BY 子句中指定的列称为 聚合键 或者 分组列。
注2： SQL 语句的顺序不能改变，也不能相互替换
1.SELECT –> 2.FROM –> 3.WHERE –> 4.GROUP BY

聚合键中包含NULL的情况

示例：按照进货单价统计数据行数

SELECT purchase_price,  COUNT(*)
    FROM product
    GROUP BY purchase_price;

注： 如果聚合键中包含NULL，在结果中会单独列成一组。

使用 WHERE 子句时 GROUP BY 的执行结果

示例：同时使用WHERE和GROUP子句

SELECT purchase_price,  COUNT(*)
    FROM product
    WHERE product_type = '衣服'
    GROUP BY purchase_price;

SQL先对数据集做WHERE过滤，后再用GROUP BY分组
注： 目前为止, SQL 语句的执行顺序为：
FROM –> WHERE –> GROUP BY –> SELECT

与聚合函数和 GROUP BY 子句有关的常见错误

常见错误 1: 在SELECT 子句中加入了聚合键之外的列
例如：

SELECT product_name, purchase_price, COUNT(*)
    FROM Product
    GROUP BY purchase_price;

注：使用 GROUP BY 子句时, SELECT 子句不能出现聚合键之外的列名

常见错误2：在 GROUP BY 子句中写了列的别名
示例：

SELECT product_type AS pt, COUNT(*)
    FROM Product
    GROUP BY pt; -- 不能使用别名

注： 在GROUP BY 子句中不能使用 SELECT 子句中定义的别名

常见错误3： GROUP BY 的结果能否排序

注： GROUP BY 子句结果的显示是无序的

常见错误 4： 在WHERE子句中使用聚合函数
示例：取出COUNT 恰好是2行的数据的组

SELECT product_type, COUNT(*)
    FROM Product
    WHERE COUNT(*) = 2 -- 这句会报错
    GROUP BY product_type;

注： 只有 SELECT 子句和 HAVING 子句（以及ORDER BY 子句）中能够使用聚合函数

为聚合结果指定条件

HAVING 子句

语法：

SELECT <列名1>, <列名2>,...
    FROM <表名>
    GROUP BY <列名1>, <列名2>, ...
    HAVING <分组结果对应的条件>

注：到目前为止的SELECT语句书写顺序为
1.SELECT –> 2.FROM –> 3.WHERE –> 4.GROUP BY –> 5.HAVING

示例：按照商品种类分组后，取出”包含数据行数为2行的组”

SELECT product_type,  COUNT(*)
    FROM product
    GROUP BY product_type
    HAVING COUNT(*) = 2;

示例2： 按照商品种类分组，取出”销售单价的平均值大于等于2500元的组”

SELECT product_type,  AVG(sale_price)
    FROM product
    GROUP BY product_type
    HAVING AVG(sale_price) >= 2500;

HAVING 子句的构成要素

HAVING 能够使用3种要素：

  • 常数
  • 聚合函数
  • GROUP BY 子句中指定的列名（聚合键）

下面这种写法就是错误的：

SELECT product_type, COUNT(*)
    FROM product
    GROUP BY product_type
    HAVING product_name = '圆珠笔'; -- 这句会报错

相对于HAVING 子句更适合写在 WHERE 子句中的条件

WHERE 子句 = 指定行所对应的条件
HAVING 子句 = 指定组所对应的条件

注： 聚合键所对应的条件不应该写在 HAVING 子句当中，而应该写在 WHERE 子句当中

对查询结果进行排序

ORDER BY 子句

语法：

SELECT <列名1>, <列名2>,...
    FROM <表名>
    ORDER BY <排序基准列1>, <排序基准列2>...

示例：按销售单价由低到高排列

SELECT product_id, product_name, sale_price, purchase_price
    FROM product
    ORDER BY sale_price;

注： ORDER BY 子句中的列名 称为排序键 。
子句的书写顺序：
1.SELECT –> 2.FROM –> 3.WHERE –> 4.GROUP BY –> 5.HAVING –> 6.ORDER BY

指定升序或降序

示例： 按照销售单价由高到低排列

SELECT product_id, product_name, sale_price, purchase_price
    FROM product
    ORDER BY sale_price DESC ;

注： 升序关键字是 ASC ， 降序关键字是 DESC
默认排序为升序。

指定多个排序键

示例： 按照销售单价和商品编号的升序进行排序

SELECT product_id, product_name, sale_price, purchase_price
    FROM product
    ORDER BY sale_price, product_id ;

NULL 的顺序

示例：按照进货单价的升序进行排列

SELECT product_id, product_name, sale_price, purchase_price
    FROM product
    ORDER BY purchase_price ;

注： 排序键中包含 NULL 时， 会在开头或末尾进行汇总。

在排序列中使用显示用的别名

示例： 在ORDER BY 子句中使用列的别名

SELECT product_id AS id, product_name, sale_price AS sp, purchase_price
    FROM product
    ORDER BY sp, id ;

注： 在ORDER BY子句中可以使用 SELECT 子句中定义的别名。

ORDER BY 子句中可以使用的列

示例： ORDER BY 子句中可以使用 SELECT 子句中未包含的列：

SELECT product_name, sale_price, purchase_price
    FROM product
    ORDER BY product_id ;

示例：ORDER BY 子句中也可以使用聚合函数

SELECT product_type, COUNT(*)
    FROM product
    GROUP BY product_type
    ORDER BY COUNT(*) ;

注： 在ORDER BY 子句中，可以使用 SELECT 子句中未使用的列和聚合函数

不要使用列编号

示例： ORDER BY 子句中使用列对应的编号

SELECT product_id, product_name, sale_price, purchase_price
    FROM product
    ORDER BY 3 DESC, 1;

注： 列编号可以使用，但是不要使用。
