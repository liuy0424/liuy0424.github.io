数据的插入 – INSERT

INSERT 语句的基本语法

语法：

INSERT INTO <表名>(列1，列2...) VALUES( 值1，值2...);

示例：

INSERT INTO PRODUCTINS (product_id, product_name, product_type, sale_price, purchase_price, regist_date) VALUES ('0001', 'T恤衫', '衣服', 1000, 500, '2009-9-20');

通常来说，一句INSERT语句插入一条数据，多条数据可以通过循环来实现。

有很多数据库也支持多行同时插入。示例：

INSERT INTO ProductIns VALUES ('0002', '打孔器', '办公用品', 500, 320, '2009-09-11'),
                              ('0003', '运动T恤', '衣服', 4000, 2800, NULL),
                              ('0004', '菜刀', '厨房用具', 3000, 2800, '2009-09-20');

列清单的省略

如果对表全列进行插入，可以省略列清单。
示例：

INSERT INTO ProductIns VALUES ('0005', '高压锅', '厨房用具', 6800, 5000, '2009-01-15');

插入 NULL

示例：

INSERT INTO ProductIns VALUES ('0006', '叉子', '厨房用具', 500, NULL, '2009-09-20');

插入默认值

要想插入默认值，需要在创建表的时候加入Default 关键字。
示例：

CREATE TABLE ProductIns
(product_id      CHAR(4)      NOT NULL,
 product_name    VARCHAR(100) NOT NULL,
 product_type    VARCHAR(32)  NOT NULL,
 sale_price      INTEGER      DEFAULT 0, -- 此处为该字段设置默认值，默认值为0。
 purchase_price  INTEGER      ,
 regist_date     DATE         ,
 PRIMARY KEY (product_id));

有了默认值字段，再向表里插入数据的时候，可以用以下两种方式插入：

--显式
INSERT INTO ProductIns (product_id, product_name, product_type, SALE_PRICE, PURCHASE_PRICE, regist_date)
    VALUES ('0007', '擦菜板', '厨房用具', DEFAULT, 790, '2009-04-28');

--隐式
INSERT INTO ProductIns (product_id, product_name, product_type, PURCHASE_PRICE, regist_date)
    VALUES ('0008', '擦菜板', '厨房用具', 790, '2009-04-28'); -- 列名和值都省略

省略INSERT语句中的列名，会自动设定该列的默认值，如果没有默认值，则会设定成NULL。如果该列设置了NOT NULL 约束，则会报错。

从其它表中复制数据

示例：先创建一个结构一样的 Productcopy表，然后使用下列命令：

INSERT INTO PRODUCTCOPY (PRODUCT_ID, PRODUCT_NAME, PRODUCT_TYPE,SALE_PRICE, PURCHASE_PRICE,REGIST_DATE)
SELECT product_id, product_name, product_type, sale_price,purchase_price, regist_date
FROM PRODUCT;

也可以插入其它表中数据的合计值。示例：

INSERT INTO PRODUCTTYPE (PRODUCT_TYPE, SUM_SALE_PRICE, SUM_PURCHASE_PRICE)
    SELECT product_type, sum(sale_price), sum(purchase_price)
    FROM PRODUCT
    GROUP BY product_type;

INSERT 语句的 SELECT 语句中，可以使用 WHERE 子句或者 GROUP BY 子句等任何 SQL 语法（但ORDER BY 子句不会产生任何效果）。

数据的删除 – DELETE

DELETE 语句的基本语法

语法： 

DELETE FROM <表名>
    WHERE <条件>;

示例：删除表中全部数据

DELETE FROM Product;

DELETE 语句删除对象并不是表或者列，而是记录（行）。

指定删除对象的 DELETE 语句

示例：删除销售单价（sale_price）大于等于4000元的数据

DELETE FROM Product
    WHERE sale_price >= 4000;

可以通过WHERE 子句指定对象条件来删除部分数据
DELETE 语句中不能使用 GROUP BY、 HAVING 和 ORDER BY 三类子句，只能使用 WHERE 子句。

数据的更新 – UPDATE

UPDATE 语句的基本语法

语法：

UPDATE <表名>
    SET <列名> = <表达式>
    WHERE <条件表达式>;

示例： 将登记日期全部更新为”2009-10-10”

UPDATE PRODUCT
    SET REGIST_DATE = '2009-10-10';

指定条件的 UPDATE 语句

示例： 将商品种类为厨房用具的记录的销售单价更新为原来的10倍

UPDATE PRODUCT
    SET SALE_PRICE = SALE_PRICE * 10
    WHERE PRODUCT_TYPE = '厨房用具';

使用 NULL 进行更新

示例：将商品编号为0008的数据（圆珠笔）的登记日期更新为NULL

UPDATE PRODUCT
    SET REGIST_DATE = NULL
    WHERE PRODUCT_ID = '0008';

使用 UPDATE 语句可以将值清空为NULL （但只限于未设置 NOT NULL 约束的列。）

多列更新

多列更新有两种方式：

-- 方式1
UPDATE Product
    SET sale_price = sale_price * 10,
        purchase_price = purchase_price / 2
    WHERE product_type = '厨房用具';

-- 方式2
UPDATE Product
    SET (sale_price, purchase_price) = (sale_price * 10, purchase_price / 2)
    WHERE product_type = '厨房用具';

方式2 有些数据库不支持，所以通常使用方式 1。

事务

什么是事务

事务(transaction) : 需要在同一个处理单元中执行一系列更新处理的集合。

创建事务

语法：

事务开始语句；
    DML语句1;
    DML语句2;
    DML语句3;
    ...
事务结束语句(COMMIT 或者 ROLLBACK);

ACID特性

DBMS的事务都遵守四种特性，简称 ACID

  • 原子性(Atomicity) : 是指所包含的更新处理要么全部执行，要么完全不执行。
  • 一致性(Consistency) : 是指事务中包含的处理要满足数据库提前设置的条件，如主键约束或者NOT NULL 约束。
  • 隔离性(Isolation) : 是指保证不同事务之间互不干扰的特性。该特性保证了事务之间不会互相嵌套。
  • 持久性(Durability) : 指的是事务结束后， DBMS能够保证该时间点的数据状态会被保存的特性。
