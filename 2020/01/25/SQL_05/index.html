视图

视图和表

视图： 从SQL角度来看，视图就是一张表。
视图和表的区别在于，视图不实际保存数据。实际上视图保存的是SQL语句。
视图的优点：
- 由于视图无需保存数据，因此可以节省存储设备的容量。
- 将频繁使用的SELECT 语句保存成视图，不用每次重新书写。

创建视图的方法

语法：

CREATE VIEW 视图名称 (<视图列名1>, <视图列名2>,...)
AS
<SELECT语句>

示例：创建ProductSum视图

CREATE VIEW ProductSum (product_type, cnt_product)
AS
    SELECT product_type, count(*)
    FROM PRODUCT
    GROUP BY product_type;

应该避免在视图的基础上创建视图（多重视图，会降低SQL性能）

视图的限制

  • 定义视图时不要使用ORDER BY 子句
  • 通过汇总得到的视图无法更新
  • 没有聚合和组合的查询构成的视图可以更新。

删除视图

语法：

DROP VIEW 视图名称 (<视图列名1>, <视图列名2>, ......);

示例：

DROP VIEW ProductSum;

子查询

子查询和视图

子查询就是将用来定义视图的 SELECT 语句 用于FROM 当中。
示例：

SELECT product_type, cnt_product
FROM (SELECT product_type, count(*) AS cnt_product
    FROM PRODUCT
    GROUP BY product_type) AS ProductSum;

子查询作为内层查询会首先执行
多层嵌套查询应尽量避免
原则上子查询必须要设定名称

标量子查询(scalar subquery)

标量子查询 ：就是返回单一值的子查询。
示例： 选取出销售单价(sale_price)高于全部商品平均单价的商品

SELECT product_id, product_name, sale_price
FROM PRODUCT
WHERE SALE_PRICE > (SELECT avg(sale_price) FROM PRODUCT); -- SELECT 子句是计算平均销量单价的标量子查询

标量子查询的书写位置

能够使用常数 或者 列名的地方，无论是 SELECT 子句、GROUP BY子句、 HAVING 子句，还是 ORDER BY 子句，几乎所有的地方都可以使用。

示例：

-- 示例1：在 SELECT中使用标量子查询
SELECT product_id, product_name, sale_price,
       (SELECT AVG(sale_price) FROM PRODUCT) AS avg_price
    FROM PRODUCT;

-- 示例2：在HAVING子句中使用标量子查询
-- 选取出按照商品种类计算出的销售单价高于全部商品的平均销售单价的商品种类。
SELECT product_type, AVG(SALE_PRICE)
    FROM PRODUCT
    GROUP BY product_type
    HAVING AVG(SALE_PRICE) > (SELECT AVG(sale_price) FROM PRODUCT);

注意事项：该子查询绝对不能返回多行结果

关联子查询

普通子查询和关联子查询的区别

关联子查询用以解决分组为单位，销售单价与平均单价相比较的场景
示例：通过关联子查询按照商品种类对平均销售单价进行比较

SELECT product_type, product_name, sale_price
    FROM PRODUCT AS P1
    WHERE SALE_PRICE > (SELECT AVG(sale_price)
                            FROM PRODUCT AS P2
                            WHERE P1.PRODUCT_TYPE = P2.PRODUCT_TYPE  -- 这句条件是关键
                            GROUP BY P1.PRODUCT_TYPE);

在细分的组内进行比较时，需要使用关联子查询。

  • 关联子查询也是用来对集合进行切分的
  • 组合条件一定要写在子查询中。原因是违反了关联名称的作用域原则。

下面这个写法就是错误的：

SELECT product_type, product_name, sale_price
    FROM Product AS P1
    WHERE P1.product_type = P2.product_type -- 关联条件不能移到子查询之外
    AND sale_price > (SELECT AVG(sale_price)
                        FROM Product AS P2
                        GROUP BY product_type);
